家，是居住者心靈的安頓之處；而對於一位空間設計師來說，個人網站就是展示專業美學與記錄思考的數位門面。

今天正式為自己建置了 **LudeLee 個人品牌網站**！這個網站兼具了部落格記錄與求職作品集的功能，設計理念源於侘寂風（Wabi-Sabi）的自然溫暖調性。以下完整記錄這次的建置過程與核心技術架構：

---

### 一、 視覺與設計理念：暖沙與細格線的侘寂美學

在視覺設計上，我參考了高見龍個人網站的精緻動態感，並融合了侘寂與現代極簡主義的色調：
*   **色系配方**：以溫暖沙色（`#f5f2eb`）為畫布底色，深炭褐（`#2b2522`）做為內文字體，並以富含泥土溫度的陶土色（`#a3704c`）作為滑鼠懸停（Hover）及標題強調節奏。
*   **雜誌感排版**：借鑑 NOVA ATELIER 與 OURA & CO. 的設計風格，在首頁使用極細灰色線條框格劃分數據與區塊，並導入 Noto Serif TC (思源宋體) 與 Playfair Display 高級襯線體，讓資訊像印刷雜誌般安靜優雅。
*   **置頂黏性導覽**：桌機端採用高度 `80px` 的置頂導覽列 (Sticky Header)，配有毛玻璃模糊效果與底線滑動 Hover 動畫；行動端則會自動轉換為底部的浮動選單，保證各裝置上的無障礙瀏覽。

---

### 二、 系統架構：零依賴與 Obsidian 的深度整合

因為我的筆記與隨筆大多儲存於 **Obsidian** 中，為了維持這個寫作習慣，網站採用了非常獨特的「Obsidian 友善」架構：

1.  **純前端原生模組架構 (No node_modules)**
    由於我的專案資料夾是在 Google Drive (我的雲端硬碟) 進行同步，安裝大量 `node_modules` 檔案容易因雲端同步鎖定而觸發寫入權限錯誤 (EPERM)。為此，我們跳過複雜的 Vite 打包依賴，改為**純原生 ES 模組與原生 CSS 變數**，無任何編譯成本，網站載入速度快如閃電，且能直接靜態部署。

2.  **動態 Markdown (.md) 解析系統**
    網頁載入了 `marked.js` 庫。這意味著我**不需要再去修改 JSON 或是寫 HTML 代碼**。
    當我想要新增或編輯網誌文章時，我只需要像平常一樣在 `posts/` 資料夾下，新增或用 Obsidian 開啟一般的 Markdown (`.md`) 檔案。檔案最上方撰寫簡單的 YAML 屬性（設定 id、標題、標籤與封面），下方直接以 Markdown 語法寫文章，網站就會在網頁端自動 fetch 並動態解析渲染！

3.  **自動防快取 (Auto Cache-Busting) 機制**
    在開發與頻繁更新過程中，瀏覽器的快取常會導致更新無法即時呈現。我們在 `main.js` 的 Fetch 請求中自動追加了時間戳記（`?t=Date.now()`），確保每次重新整理網頁時，瀏覽器都能抓取到最新寫好的文章與履歷。

---

### 三、 部署與自動化：雙腳本一鍵部署 GitHub Pages

網頁部署於免費的 **GitHub Pages** 靜態伺服器。為了讓未來的寫作流程更加傻瓜化，我們在專案根目錄設計了自動推送工具：

*   **`deploy.bat` (啟動外殼)**：雙擊即可執行，內部為純英文 ASCII 代碼，避免了 Windows 命令提示字元 (CMD) 在 UTF-8 與 ANSI 轉換時，將中文註解解析為無效指令的先天編碼 Bug。
*   **`deploy.ps1` (執行主體)**：以 PowerShell 編寫並儲存為符合微軟規範的 `UTF-8 with BOM` 編碼。它會詢問您本次的更新說明，執行 `git add/commit/push`，並在成功後**自動打開您的瀏覽器至 GitHub Actions 進度頁面**。當 Actions 頁面中的黃色圈圈轉為綠色勾勾時，便能確切知道網站更新已完成。
